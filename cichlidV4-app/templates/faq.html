This website provides access to sample and sequencing information for the East-African cichlid project carried out in Richard Durbin's group.
Below are answers to Frequently Asked Questions. If you cannot find the information you are looking for is not present, send us an email here

Which data can be accessed via this website?

Whole table data can be accessed using the 'Select a table' button.
Specific 'project', 'individual', 'material', 'sample' and 'file' data are available following the links in the whole table displayed or entering one or more query terms in the text boxes on the home page.
Extra information will be displayed upon clicking on the '+' sign above the tables.
Visibility of one or more columns can be toggled on, or off, using the 'add/remove columns' button available upon every displayed table.

Can I access the data programmatically?

Yes.
The URL for the web page have three type of structure:
- https://darwin-tracking.sanger.ac.uk/cichlid/api/1.1/{table}/web (for example: "https://darwin-tracking.sanger.ac.uk/cichlid/api/1.1/project/web")
- https://darwin-tracking.sanger.ac.uk/cichlid/api/1.1/{table}/{table_id}/web (for example: "https://darwin-tracking.sanger.ac.uk/cichlid/api/1.1/project/1/web")
- https://darwin-tracking.sanger.ac.uk/cichlid/api/1.1/{table}/name/{name}/web (for example: "https://darwin-tracking.sanger.ac.uk/cichlid/api/1.1/individual/name/D01-A03/web")

The data can be accessed as a json format by replacing the suffix 'web' by 'json'
For example, using the second example above: "curl https://darwin-tracking.sanger.ac.uk/cichlid/api/1.1/project/1/json" will display a json file with this structure:
("project:1":
project:[
{project_field1: project_data1, project_field2: project_data2, .....}
]
material:[
{material_field1: material1_data1, material_field2: material1_data2, .....},
{material_field1: material2_data1, material_field2: material2_data2, .....},...
]
individual:[
{individual_field1: individual1_data1, individual_field2: individual1_data2, .....},
{individual_field1: individual2_data1, individual_field2: individual2_data2, .....},...
]
sample:[
{sample_field1: sample1_data1, sample_field2: sample1_data2, .....},
{sample_field1: sample2_data1, sample_field2: sample2_data2, .....},...
]
file:[
{file_field1: file1_data1, file_field2: file1_data2, .....},
{file_field1: file2_data1, file_field2: file2_data2, .....},...
]
)
Similarly, the call for the third example will be: "curl https://darwin-tracking.sanger.ac.uk/cichlid/api/1.1/individual/name/D01-A03/json".

- Several 'individual' and 'sample' names can be inserted: they need to be separated by a comma, or "%2C", with no space.
- For 'species' and 'location': add the flag '-L' to instruct curl to follow the links. Example: "curl -L http://127.0.0.1:5000/api/1.1/location/name/Liwonde/json". Spaces in the names have to be replaced by "%20".
- Species queries accept partial name. For example: "curl -L http://127.0.0.1:5000/api/1.1/species/name/Buccoc/json" is the equivalent of "curl -L http://127.0.0.1:5000/api/1.1/species/name/Buccochromis/json".
- Note that the tag 'all' can be added to obtain more information, equivalent to the '+' sign on the web pages, for tables 'individual', 'lane', 'material', 'species' and 'provider'. Example: "curl -L http://127.0.0.1:5000/api/1.1/species/all/json".
- The tag 'all' can also be used in conjonction with '.../individual/{id}/json', '.../material/{id}/json' and '.../sample/{id}/json'

How could I filter the display?

In addition to the text boxes available on the home page, a 'Search' box is available on every tab displayed.
The search will take place in all columns of the displayed table. The term(s) provided will be match partially.

How can I download the data from the website?

The data from the tables displayed on each page can be downloaded in 'tsv' (tab-separated format), excel and pdf format using the button displayed above the tables.
Note that the downloaded pages will reflect the filtering applied via the 'Search' box but not the 'add/remove columns' button.
Data can also be accessed programmatically. See the FAQ page for details.

How can I add new data to the database?

In order to add data to the database, you first need to be logged in. For that click on the 'Login' button located on the top-right side of the page.
Then you will be able to download a template, in tab-separated format, or fill the provided form.
After uploading the filled template or submitting the completed form, we will review the data before adding them to the database in a timely manner.
